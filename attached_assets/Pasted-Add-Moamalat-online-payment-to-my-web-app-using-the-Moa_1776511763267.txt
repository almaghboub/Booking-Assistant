Add Moamalat online payment to my web app using the Moamalat LightBox integration.

Requirements:
- Use a backend route to generate the Moamalat payment config securely.
- Do NOT generate the secure hash in the frontend.
- Use environment variables for secrets.
- Use the Moamalat test LightBox script in development:
  https://tnpg.moamalat.net:6006/js/lightbox.js
- Keep production-ready structure so I can later switch to:
  https://npg.moamalat.net:6006/js/lightbox.js

Moamalat integration details:
- Required fields:
  MID
  TID
  AmountTrxn
  MerchantReference
  TrxDateTime
  SecureHash
- AmountTrxn must be in the smallest currency unit. 1 LYD = 1000.
- TrxDateTime format: yyyyMMddHHmm
- Use callbacks:
  completeCallback
  errorCallback
  cancelCallback

Secure hash rules:
- Generate SHA-256 HMAC in uppercase hex.
- Use the merchant secret key as a hex-decoded key.
- For the LightBox request hash, use these fields only:
  Amount
  DateTimeLocalTrxn
  MerchantId
  MerchantReference
  TerminalId
- Sort field names ascending.
- Build the string like:
  Amount=...&DateTimeLocalTrxn=...&MerchantId=...&MerchantReference=...&TerminalId=...
- Then compute HMAC-SHA256 and return uppercase hex.
